# FastPick 渐进式加载优化计划

## 当前实现分析

项目已有良好的渐进式加载基础架构：

### 图片列表加载
- 阶段A：快速扫描，仅获取文件名，立即显示前100张
- 阶段B：后台增量加载剩余图片，每200张一个批次
- 阶段C：后台读取完整元数据

### 预览图加载
- **按需加载模式**：直接加载普通预览（1920x1080）
- **分级预取模式**：
  - 先加载快速缩略图（256px缓存或1024px）
  - 防抖150ms后加载高清原图

---

## 分级预取模式专项优化

### [ ] 1. 双文件智能选择 - 优先使用JPG作为缩略图
**优先级**: P0
**描述**: 当前实现仅使用`photoItem.DisplayPath`，对于JPG+RAW双文件，应优先使用JPG获取缩略图，速度更快、质量更好

**优化内容**:
- 在`LoadQuickPreviewAsync`中，优先尝试JPG路径
- 如果JPG不存在或失败，再回退到RAW
- 相应调整高清加载时的文件选择逻辑

**成功标准**: 双文件场景下，快速缩略图加载时间减少30%以上

---

### [ ] 2. 防抖策略优化 - 动态调整 + 并行启动
**优先级**: P0
**描述**: 优化防抖机制，在保证流畅性的同时减少等待时间

**优化内容**:
- 防抖时间可配置（默认100-150ms）
- 快速缩略图加载时间超过阈值（如200ms）时，提前启动高清加载
- 根据用户浏览速度动态调整防抖时间（快速浏览时缩短防抖）

**成功标准**: 用户停留查看时，高清图在防抖后立即显示；快速切换时不浪费资源

---

### [ ] 3. 多级分辨率策略 - 根据屏幕自适应
**优先级**: P1
**描述**: 固定的1024px和1920x1080不一定最优，应根据窗口大小和DPI动态调整

**优化内容**:
- 快速缩略图：根据窗口宽度的1/2或屏幕DPI计算（如DPI 2.0时用2048px）
- 高清图：根据实际预览区域尺寸计算，避免不必要的高分辨率
- 提供"质量优先"和"速度优先"选项

**成功标准**: 在不同分辨率屏幕下都能获得最佳的速度/质量平衡

---

### [ ] 4. 缓存优化 - 快速缩略图 + 高清图双缓存
**优先级**: P1
**描述**: 实现图片缓存，避免重复解码

**优化内容**:
- LRU内存缓存：最近查看的20-50张图片（快速+高清）
- 磁盘缓存：将解码后的缩略图保存到临时目录
- 缓存命中率统计
- 应用退出或内存压力时自动清理

**成功标准**: 重复访问图片时，加载速度提升80%以上

---

### [ ] 5. 智能预加载 - 相邻图片预加载
**优先级**: P1
**描述**: 预测用户即将查看的图片，提前加载快速缩略图

**优化内容**:
- 预加载当前图片前后各2-3张的快速缩略图
- 高清图仅在用户停留后才预加载
- 可配置预加载数量
- 切换图片时立即取消过时的预加载

**成功标准**: 用户快速切换时，80%的图片已有快速缩略图

---

### [ ] 6. 快速加载失败处理 - 直接启动高清加载
**优先级**: P2
**描述**: 当前实现中，快速加载失败会回退到普通加载，再走防抖流程，浪费时间

**优化内容**:
- 快速加载失败时，直接启动高清加载，不走防抖
- 或者提供选项：快速加载失败时是否立即显示普通预览

**成功标准**: 快速加载失败的场景下，高清图显示时间减少150ms

---

### [ ] 7. RAW内嵌预览优化 - 作为快速加载备选
**优先级**: P2
**描述**: 对于只有RAW的文件，快速加载可以尝试RAW内嵌预览

**优化内容**:
- 快速加载时，RAW文件先尝试内嵌预览
- 内嵌预览失败再尝试系统缩略图
- 内嵌预览成功则跳过1024px步骤

**成功标准**: 纯RAW文件的快速加载成功率提升

---

### [ ] 8. 加载优先级管理 - 确保当前图片优先
**优先级**: P1
**描述**: 建立任务优先级机制，确保用户当前需要的资源优先加载

**优化内容**:
- 当前图片快速加载：最高优先级
- 当前图片高清加载：高优先级
- 相邻图片预加载：中等优先级
- 缩略图批量生成：低优先级
- 元数据加载：最低优先级
- 实现优先级队列，支持动态调整

**成功标准**: 当前图片加载不受后台任务影响

---

### [ ] 9. 内存优化 - 智能释放资源
**优先级**: P1
**描述**: 优化内存使用，防止长时间使用后内存占用过高

**优化内容**:
- 监听应用内存压力事件
- 当内存占用过高时，自动清理LRU缓存
- 实现图片资源的显式释放
- 大图缩小显示时，保留合适尺寸的版本

**成功标准**: 长时间浏览后，内存占用保持在合理范围内（< 1GB）

---

### [ ] 10. 缩略图生成优化 - 并行生成 + 质量自适应
**优先级**: P2
**描述**: 优化缩略图生成策略，根据屏幕分辨率和显示需求调整

**优化内容**:
- 根据屏幕DPI自动调整缩略图尺寸
- 缩略图批量并行生成（限制并发数）
- 支持不同质量级别的缩略图
- 优先生成当前可见区域的缩略图

**成功标准**: 缩略图区域滚动流畅，无明显卡顿

---

### [ ] 11. RAW文件处理优化 - 更智能的降级策略
**优先级**: P2
**描述**: 优化RAW文件预览加载，在质量和速度间取得更好平衡

**优化内容**:
- 优先使用JPG作为预览（如果存在）
- RAW内嵌预览失败时的智能重试机制
- 根据相机型号优化RAW解码参数
- 提供RAW解码质量/速度权衡选项

**成功标准**: RAW文件预览加载成功率提升，加载时间缩短

---

### [ ] 12. 性能监控与分析
**优先级**: P2
**描述**: 添加性能监控，帮助识别瓶颈和持续优化

**优化内容**:
- 记录各阶段加载时间
- 统计缓存命中率
- 记录用户浏览模式数据
- 提供性能日志导出功能

**成功标准**: 能够通过数据分析识别性能瓶颈
