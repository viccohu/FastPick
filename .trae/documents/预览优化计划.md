# 预览优化计划 - 实施计划

## 概述
优化预览加载和缓存机制，减少内存占用，提升用户体验。

## 任务清单

### [ ] 任务 1: 修改 PreviewService，默认只加载到 L2
- **优先级**: P0
- **依赖**: 无
- **描述**:
  - 修改 `LoadPreviewAsync` 方法，默认目标级别改为 `QuickPreview` (L2)
  - 保留 `LoadProgressiveAsync` 方法供特殊情况调用
  - 快速翻页模式保持只加载 L2
- **成功标准**: 默认预览加载时不加载 L3，只加载到 L2
- **测试要求**:
  - `programmatic`: 点击缩略图预览时，日志显示只加载 L1 和 L2，不加载 L3
  - `human-judgement`: 预览显示速度明显提升，没有卡顿

---

### [ ] 任务 2: 添加 L3 延迟加载机制（放大时触发）
- **优先级**: P0
- **依赖**: 任务 1
- **描述**:
  - 在 MainPage 中检测放大行为
  - 当用户放大超过 100%（或某个阈值）时，触发 L3 加载
  - L3 加载完成后，无感切换显示
  - 当缩放回到适应窗口或 100% 以下时，切换回 L2
  - 需要添加防抖机制，避免频繁触发 L3 加载
- **成功标准**: 放大时无感加载 L3，缩小时切换回 L2
- **测试要求**:
  - `programmatic`: 放大超过阈值时，日志显示 L3 开始加载；缩小时，日志显示切换回 L2
  - `human-judgement`: 放大时图片清晰度逐渐提升，没有明显卡顿

---

### [ ] 任务 3: 优化缓存机制，只缓存到 L2
- **优先级**: P1
- **依赖**: 无
- **描述**:
  - 调整 PreviewCacheManager 的 L3 缓存配置，大幅减小或禁用
  - 建议：L3 缓存设置为 0 或只保留 1-2 张
  - 预加载机制也相应调整，只预加载 L2
  - L3 只在需要时（放大）才加载，不缓存
- **成功标准**: L3 缓存大幅减小或禁用，内存占用降低
- **测试要求**:
  - `programmatic`: 内存使用监控显示 L3 缓存占用大幅降低
  - `human-judgement`: 长时间使用后，内存增长速度明显减缓

---

### [ ] 任务 4: 优化预加载机制
- **优先级**: P1
- **依赖**: 任务 3
- **描述**:
  - PreDecodeManager 只预加载 L2，不预加载 L3
  - 保持当前的滑动窗口预加载策略（±2 张）
  - 确保预加载任务优先级合理
- **成功标准**: 预加载只加载 L2，提升翻页体验
- **测试要求**:
  - `programmatic`: 预加载日志显示只加载 L2
  - `human-judgement`: 左右翻页时，L2 图片快速显示

---

### [ ] 任务 5: 内存管理优化和检查
- **优先级**: P1
- **依赖**: 无
- **描述**:
  - 检查当前预览代码中的内存管理
  - 确保 PhotoItem 在不需要时能正确释放 L3 引用
  - 当切换回 L2 时，确保 L3 的 BitmapImage 被正确释放
  - 添加定期的 GC 提示（可选）
- **成功标准**: 内存管理无泄漏，图片切换时资源正确释放
- **测试要求**:
  - `programmatic`: 在内存压力测试下，没有内存泄漏
  - `human-judgement`: 长时间使用后，内存不会无限增长

---

## 实施顺序
1. 先完成任务 1（默认加载 L2）
2. 然后完成任务 2（放大加载 L3）
3. 接着完成任务 3（缓存优化）和任务 4（预加载优化）
4. 最后完成任务 5（内存管理检查）

## 预期效果
- 内存占用大幅降低（主要来自减少 L3 缓存）
- 预览加载速度显著提升（只加载到 L2）
- 放大时仍能获得高清晰度（L3 延迟加载）
- 翻页体验流畅（L2 预加载）
