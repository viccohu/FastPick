# 多线程解码去重方案

## 一、当前实现分析

### 1.1 现有去重机制

```csharp
// GetThumbnailAsync 方法中的去重逻辑
var tcs = new TaskCompletionSource<BitmapImage?>();
if (_pendingLoads.TryAdd(filePath, tcs))
{
    // 第一个线程：执行解码
    await _throttler.WaitAsync(cancellationToken);
    var wicThumbnail = await GenerateWicThumbnailAsync(filePath, cancellationToken);
    tcs.SetResult(wicThumbnail);
}
else
{
    // 后续线程：等待第一个线程的结果
    if (_pendingLoads.TryGetValue(filePath, out var existingTcs))
    {
        return await existingTcs.Task;
    }
    return null;
}
```

### 1.2 现有机制的问题

#### 问题 1：TryAdd 和 TryGetValue 之间存在竞态条件

```csharp
// 线程 A：TryAdd 成功
if (_pendingLoads.TryAdd(filePath, tcs))  // 线程 A 进入

// 线程 B：TryAdd 失败，进入 else 分支
else
{
    // 此时线程 A 可能还没完成 TryAdd，或者已经完成并移除
    if (_pendingLoads.TryGetValue(filePath, out var existingTcs))
    {
        return await existingTcs.Task;  // 可能获取到 null
    }
    return null;  // 错误！应该返回解码结果或重新加载
}
```

#### 问题 2：异常传播不完整

```csharp
catch (OperationCanceledException)
{
    tcs.SetCanceled();  // 正确传播取消
    throw;
}
catch (Exception ex)
{
    tcs.SetResult(null);  // 问题：后续线程收到 null，不知道是失败还是真的没有图片
    return null;
}
```

#### 问题 3：没有超时机制

- 如果 `GenerateWicThumbnailAsync` 卡住，所有等待的线程都会无限等待
- 没有超时保护

#### 问题 4：取消后无法重试

- 如果第一个线程被取消，后续线程收到 `TaskCanceledException`
- 没有重试机制，需要用户重新触发加载

---

## 二、解决方案

### 2.1 方案一：改进现有去重机制（推荐）

**核心改进**：
1. 使用 `GetOrAdd` 模式确保原子性
2. 添加超时机制
3. 添加失败重试逻辑
4. 正确传播异常

```csharp
private readonly ConcurrentDictionary<string, Task<BitmapImage?>> _pendingLoads = new();
private readonly TimeSpan _loadTimeout = TimeSpan.FromSeconds(30);

public async Task<BitmapImage?> GetThumbnailAsync(PhotoItem photoItem, CancellationToken cancellationToken = default)
{
    var filePath = photoItem.DisplayPath;
    
    // 1. 检查内存缓存
    if (_thumbnailCache.TryGetValue(filePath, out var weakRef) && 
        weakRef.TryGetTarget(out var cachedImage))
    {
        return cachedImage;
    }
    
    // 2. 检查本地缓存
    var localCacheImage = await GetFromLocalCacheAsync(filePath, cancellationToken);
    if (localCacheImage != null)
    {
        _ = Task.Run(() => AddToCacheAsync(filePath, localCacheImage, photoItem), CancellationToken.None);
        return localCacheImage;
    }
    
    // 3. 使用 GetOrAdd 确保原子性
    var loadTask = _pendingLoads.GetOrAdd(filePath, _ => LoadThumbnailWithTimeoutAsync(filePath, photoItem, cancellationToken));
    
    try
    {
        // 使用带超时的等待
        using var cts = CancellationTokenSource.CreateLinkedTokenSource(cancellationToken);
        cts.CancelAfter(_loadTimeout);
        
        var result = await loadTask.WaitAsync(cts.Token);
        
        // 如果成功，添加到缓存
        if (result != null)
        {
            photoItem.Thumbnail = result;
        }
        
        return result;
    }
    catch (TimeoutException)
    {
        // 超时，移除任务允许重试
        _pendingLoads.TryRemove(filePath, out _);
        DebugService.WriteLine($"[GetThumbnail] 加载超时: {Path.GetFileName(filePath)}");
        return null;
    }
    catch (OperationCanceledException) when (cancellationToken.IsCancellationRequested)
    {
        // 用户取消，不移除任务（让其他等待的线程继续）
        throw;
    }
    catch (Exception ex)
    {
        // 其他异常，移除任务允许重试
        _pendingLoads.TryRemove(filePath, out _);
        DebugService.WriteLine($"[GetThumbnail] 加载失败: {Path.GetFileName(filePath)}, 错误: {ex.Message}");
        return null;
    }
}

private async Task<BitmapImage?> LoadThumbnailWithTimeoutAsync(string filePath, PhotoItem photoItem, CancellationToken cancellationToken)
{
    try
    {
        await _throttler.WaitAsync(cancellationToken);
        try
        {
            var result = await GenerateWicThumbnailAsync(filePath, cancellationToken);
            if (result != null)
            {
                await AddToCacheAsync(filePath, result, photoItem, CancellationToken.None);
            }
            return result;
        }
        finally
        {
            _throttler.Release();
        }
    }
    finally
    {
        // 延迟移除，确保其他等待的线程能获取结果
        await Task.Delay(100);
        _pendingLoads.TryRemove(filePath, out _);
    }
}
```

### 2.2 方案二：使用 AsyncLazy（更简洁）

```csharp
// 使用 AsyncLazy 实现懒加载 + 去重
private readonly ConcurrentDictionary<string, AsyncLazy<BitmapImage?>> _pendingLoads = new();

public async Task<BitmapImage?> GetThumbnailAsync(PhotoItem photoItem, CancellationToken cancellationToken = default)
{
    var filePath = photoItem.DisplayPath;
    
    // 1. 检查缓存...
    
    // 2. 使用 AsyncLazy 确保只加载一次
    var lazyLoad = _pendingLoads.GetOrAdd(filePath, _ => new AsyncLazy<BitmapImage?>(
        () => LoadThumbnailInternalAsync(filePath, photoItem, cancellationToken)));
    
    try
    {
        return await lazyLoad.Task.WaitAsync(TimeSpan.FromSeconds(30), cancellationToken);
    }
    catch (TimeoutException)
    {
        _pendingLoads.TryRemove(filePath, out _);
        return null;
    }
}

// AsyncLazy 实现
public class AsyncLazy<T>
{
    private readonly Lazy<Task<T>> _lazyTask;
    
    public AsyncLazy(Func<Task<T>> taskFactory)
    {
        _lazyTask = new Lazy<Task<T>>(() => Task.Run(taskFactory));
    }
    
    public Task<T> Task => _lazyTask.Value;
}
```

### 2.3 方案三：使用信号量 + 状态机（最可靠）

```csharp
private enum LoadState
{
    NotStarted,
    Loading,
    Succeeded,
    Failed
}

private class LoadContext
{
    public LoadState State { get; set; } = LoadState.NotStarted;
    public BitmapImage? Result { get; set; }
    public Exception? Error { get; set; }
    public SemaphoreSlim Lock { get; } = new(1, 1);
    public List<TaskCompletionSource<BitmapImage?>> Waiters { get; } = new();
}

private readonly ConcurrentDictionary<string, LoadContext> _loadContexts = new();

public async Task<BitmapImage?> GetThumbnailAsync(PhotoItem photoItem, CancellationToken cancellationToken = default)
{
    var filePath = photoItem.DisplayPath;
    
    // 1. 检查缓存...
    
    // 2. 获取或创建加载上下文
    var context = _loadContexts.GetOrAdd(filePath, _ => new LoadContext());
    
    await context.Lock.WaitAsync(cancellationToken);
    try
    {
        switch (context.State)
        {
            case LoadState.Succeeded:
                return context.Result;
                
            case LoadState.Failed:
                // 允许重试
                context.State = LoadState.NotStarted;
                goto case LoadState.NotStarted;
                
            case LoadState.Loading:
                // 等待正在进行的加载
                var tcs = new TaskCompletionSource<BitmapImage?>();
                context.Waiters.Add(tcs);
                context.Lock.Release();
                
                try
                {
                    return await tcs.Task.WaitAsync(TimeSpan.FromSeconds(30), cancellationToken);
                }
                finally
                {
                    context.Lock.Wait();
                    context.Waiters.Remove(tcs);
                }
                
            case LoadState.NotStarted:
                context.State = LoadState.Loading;
                break;
        }
    }
    finally
    {
        context.Lock.Release();
    }
    
    // 执行加载
    try
    {
        var result = await LoadThumbnailInternalAsync(filePath, photoItem, cancellationToken);
        
        await context.Lock.WaitAsync();
        try
        {
            context.Result = result;
            context.State = result != null ? LoadState.Succeeded : LoadState.Failed;
            
            // 通知所有等待者
            foreach (var waiter in context.Waiters)
            {
                waiter.SetResult(result);
            }
            context.Waiters.Clear();
        }
        finally
        {
            context.Lock.Release();
        }
        
        return result;
    }
    catch (Exception ex)
    {
        await context.Lock.WaitAsync();
        try
        {
            context.Error = ex;
            context.State = LoadState.Failed;
            
            foreach (var waiter in context.Waiters)
            {
                waiter.SetException(ex);
            }
            context.Waiters.Clear();
        }
        finally
        {
            context.Lock.Release();
        }
        
        throw;
    }
    finally
    {
        _loadContexts.TryRemove(filePath, out _);
    }
}
```

## 六、总结

当前代码**确实存在多线程解码同一张图片的风险**，主要体现在：

1. **TryAdd 和 TryGetValue 之间存在竞态条件**
2. **没有超时机制**
3. **异常传播不完整**
4. **取消后无法重试**

推荐使用**方案三（状态机）**来解决这些问题，它提供了：
- 完全避免竞态条件
- 超时保护
- 失败重试
- 正确的异常传播
