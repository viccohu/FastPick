# 增量加载优化方案

## 当前问题分析

### 问题表现
快速拉动滚动条到未加载区域时，滚动条会"卡住"，等待增量加载完成后才能继续。

### 根本原因
1. **每批次 UI 线程阻塞**：`LoadIncrementalAsync` 每 200 张就在 UI 线程调用 `PhotoItems.Add(item)` 和 `ApplyFilter()`
2. **频繁集合变更通知**：触发大量 `CollectionChanged` 和 `PropertyChanged` 事件
3. **筛选计算开销**：每次都重新计算 `FilteredPhotoItems`

## 优化方案

### 方案一：增加批次大小 + 减少 ApplyFilter 调用
- 将 `IncrementalBatchSize` 从 200 增加到 500-1000
- 减少 `ApplyFilter()` 调用频率（例如每 3-5 批次才调用一次）
- 优点：改动最小
- 缺点：仍然有 UI 线程阻塞

### 方案二：后台准备 + 批量添加
- 在后台线程准备好所有 PhotoItem 的基础信息和评级
- 然后在 UI 线程一次性或更大批量添加到集合
- 优点：减少 UI 线程阻塞次数
- 缺点：内存占用稍高

### 方案三：预分配 + 延迟筛选
- 初始加载时预分配 PhotoItems 的容量
- 延迟 ApplyFilter 到所有图片加载完成后
- 优点：最彻底的优化
- 缺点：改动较大

## 推荐方案：方案一 + 方案二的混合
1. 增加批次大小到 500
2. 每 2 批次才调用一次 ApplyFilter()
3. 保持 Task.Delay(50ms) 给 UI 喘息
