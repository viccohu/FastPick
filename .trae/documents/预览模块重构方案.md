# 预览模块重构方案

## 一、方案概述

本方案基于提供的技术文档建议，结合现有 FastPick 项目架构，重新设计整个预览模块，实现：

* 顺序浏览时的瞬时切换效果

* 跳跃预览时的三级渐进加载

* 快速翻页时的智能响应机制

#### 3.2.2 三级加载管理器 (ProgressiveLoadManager)

**功能**：实现三级渐进加载架构

**L1 - 缩略图**：< 5ms（复用现有 ThumbnailService）

* 来源：`ThumbnailService.GetCachedThumbnailAsync()` 或 `ThumbnailService.GetThumbnailAsync()`

* 访问方式：通过 `photoItem.Thumbnail` 或 `_thumbnailService.GetCachedThumbnailAsync(item)`

* 用途：跳跃式预览的占位图

**L2 - 快速大图**：\~30-50ms

* 来源：WIC DCT 下采样解码 (1/4 或 1/8)

* 用途：过渡显示，消除模糊感

**L3 - 原图/细节**：150ms+

* 来源：全像素解码

* 用途：最终高清显示

```csharp
public class ProgressiveLoadManager
{
    // 快速大图下采样比例
    private const double QuickPreviewScale = 0.125; // 1/8
    
    // 引用现有 ThumbnailService
    private readonly ThumbnailService _thumbnailService;
    
    public ProgressiveLoadManager(ThumbnailService thumbnailService)
    {
        _thumbnailService = thumbnailService;
    }
    
    // 三级加载入口
    public async Task<BitmapImage?> LoadProgressiveAsync(
        PhotoItem item,
        PreviewQualityLevel targetLevel,
        CancellationToken cancellationToken)
    {
        // L1: 尝试从 ThumbnailService 获取缩略图（最快）
        if (targetLevel >= PreviewQualityLevel.Thumbnail)
        {
            var thumbnail = await _thumbnailService.GetCachedThumbnailAsync(item, cancellationToken);
            if (thumbnail != null)
            {
                return thumbnail; // 立即返回 L1
            }
        }
        
        // L2: 加载快速大图
        if (targetLevel >= PreviewQualityLevel.QuickPreview)
        {
            var quickPreview = await LoadQuickPreviewAsync(item, cancellationToken);
            if (quickPreview != null)
            {
                return quickPreview; // 返回 L2，后台继续加载 L3
            }
        }
        
        // L3: 加载原图
        if (targetLevel >= PreviewQualityLevel.FullResolution)
        {
            return await LoadFullResolutionAsync(item, cancellationToken);
        }
        
        return null;
    }
```

## 五、关键技术点

### 5.1 L1 缩略图复用 ThumbnailService

```csharp
// 直接使用现有 ThumbnailService 的功能
// 1. 内存缓存：_thumbnailCache (WeakReference + LRU)
// 2. 本地缓存：磁盘缓存文件夹
// 3. 获取方式：
var thumbnail = await _thumbnailService.GetCachedThumbnailAsync(item, ct);
// 或直接访问已缓存的：
var cachedThumbnail = item.Thumbnail as BitmapImage;
```

### 5.2 WIC JPEG 下采样（L2 快速大图）

```csharp
// 在 IWICBitmapSourceTransform 中设置输出尺寸为 1/8
// WIC 会在解码过程中直接丢弃不需要的 DCT 系数
// 效果：速度提升 3-5 倍，内存占用降低 90% 以上
```

### 5.3 智能加载策略决策

```
快速翻页 → 检查内存缓存
    ├─ L3 缓存 → 直接显示
    ├─ L2 缓存 → 显示 L2，后台加载 L3
    └─ 无缓存 → 通过 ThumbnailService 获取 L1 缩略图
            └─ 后台异步加载 L2→L3
```

### 5.4 预解码窗口管理

```
当前位置: n
预解码范围: n-2, n-1, n, n+1, n+2
预解码顺序: n+1 (优先级最高) → n-1 → n+2 → n-2
预解码内容: L2 快速大图 + L3 原图
```

## 六、预期效果

1. **顺序浏览**：切换时直接展示，无延迟感
2. **跳跃预览**：先用缩略图占位，再逐级无感展示为原图
3. **快速翻页**：智能响应内存内容，优先显示已有缓存

## 七、风险与缓解

| 风险           | 影响 | 概率 | 缓解措施                      |
| ------------ | -- | -- | ------------------------- |
| 内存占用过高       | 高  | 中  | LRU 缓存策略 + 内存超限清理         |
| 预解码占用过多 CPU  | 中  | 低  | 智能检测 CPU 负载，空闲时才预解码       |
| 快速翻页时任务取消不及时 | 中  | 中  | 使用 CancellationToken 及时取消 |
| RAW 文件解码性能问题 | 高  | 中  | 优先使用内嵌预览，降级策略完善           |
