# 标题栏进度条实现计划

## 概述
在标题栏添加一个进度条，用于显示后台解码、删除、导出等操作的进度。

---

## [ ] 任务 1: 在 MainViewModel 中添加进度管理功能
- **Priority**: P0
- **Depends On**: None
- **Description**: 
  - 添加进度显示状态属性（IsProgressVisible, ProgressTitle, ProgressValue）
  - 添加后台解码进度管理（按批次虚拟进度，不深入业务代码）
  - 添加删除进度管理
  - 添加导出进度管理
  - 确保线程安全，使用 Dispatcher 更新 UI
- **Success Criteria**:
  - MainViewModel 新增属性可以正确绑定到 UI
  - 后台解码按批次更新进度（不深入业务代码）
  - 删除和导出进度正确更新
- **Test Requirements**:
  - `programmatic` TR-1.1: 检查新增的 ViewModel 属性是否正确实现
  - `human-judgement` TR-1.2: 验证进度更新不会导致 UI 卡顿

---

## [ ] 任务 2: 在 MainPage.xaml 中添加进度条 UI
- **Priority**: P0
- **Depends On**: Task 1
- **Description**:
  - 在标题栏 Grid 中添加进度条容器
  - 高度 35px，上下居中
  - 背景使用 HoverBackgroundBrush
  - 文字和进度条颜色使用灰色（TextSecondaryBrush）
  - 包含居中的行为名称文本
  - 包含进度条控件
  - 初始状态隐藏
- **Success Criteria**:
  - UI 布局符合要求
  - 样式正确应用
  - 绑定正确连接到 ViewModel
- **Test Requirements**:
  - `programmatic` TR-2.1: 验证 XAML 结构正确
  - `human-judgement` TR-2.2: 检查 UI 视觉效果是否符合要求

---

## [ ] 任务 3: 更新 ThumbnailService 支持后台解码进度通知
- **Priority**: P1
- **Depends On**: Task 1
- **Description**:
  - 添加进度回调接口到 StartBackgroundDecodingAsync
  - 按批次通知进度（虚拟进度，不影响性能）
  - 不深入嵌套在业务代码中
  - 任务完成时通知结束
- **Success Criteria**:
  - 后台解码按批次通知进度
  - 不影响解码性能
- **Test Requirements**:
  - `programmatic` TR-3.1: 验证进度回调被正确调用
  - `human-judgement` TR-3.2: 确认解码性能没有明显下降

---

## [ ] 任务 4: 更新删除操作支持进度显示

- **Success Criteria**:
